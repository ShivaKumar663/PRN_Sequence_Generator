

PRN sequence
PRN (Pseudo Random Noise) Sequence is a deterministic digital sequence that looks random but is actually generated by a known algorithm.
IRNSS PRN CODES
IRNSS utilizes Gold codes for the SPS (Standard Positioning Service) signal. The codes are selected based on the auto correlation and cross-correlation properties. The codes are generated using Linear Feedback Shift Registers. The length of each code is 1023 chips. The code is chipped at 1.023 Mcps.
SPS Code Generation 
SPS code generation, the two polynomials G1 and G2 are as defined below 
G1: X10 + X3+ 1
G2: X10 +X9 + X8 + X6 +X3 + X2 + 1.

Polynomial G1 and G2 are similar to the ones used by GPS C/A signal. The G1 and G2 generators are realized by using 10 bits Maximum Length Feedback Shift Registers (MLFSR). The initial state of G2 provides the chip delay. The G1 register is initialized with all bits as 1. G1 and G2 are XOR’ed for the generation of the final 1023 chip long PRN sequence. The time period of the PRN sequence is 1 millisecond.

As mentioned earlier PRN sequence is generated using Linear Feedback Shift Registers (LFSRs) and XOR gates. The process involves 'tapping' specific positions of the shift registers according to a generator polynomial. These tapped bits are XORed together to form a feedback bit, which is then shifted into the first position of the register at each clock cycle. To create the final 1023-chip Gold code, the outputs from two separate shift registers (G1 and G2) are XORed together to produce the unique satellite signature. Designing of PRN Sequence Generator in Verilog for S-SPS
The design consists 3 modules Top Module, PRN ID Selection and PRN ID Generator. The design analysis will be done on Basys 3 FPGA that belongs to Artix7 family. 
The Top Module holds these two modules and all the ports will be interconnected to each other.
The clock period which is used for these modules will be 50Mhz which is 20ns that is derived from the 100Mhz clk. All the module in the design works on positive edge of the clock and all module in the design contains synchronous reset.

MODULE - PRN ID SELECTOR
This module is used to give initial condition of the G2 register based on the id we are selecting.
I/O Ports:
Inputs
id_clk: Clock input to the design which supplies 50Mhz which is 20ns.
id_rst: The reset here is a active high reset which activates on the active high signal.
id: In this is the port the inputs are ranging from 0-15 we can give Id  from 1-14
Outputs
rd_en: rd_en signal goes high when id_rst goes low, indication that g2_ini can be initialized with the required value.
 g2_ini: This output shows the assigned initial condition based on the selected id.


When the particular ID is given as an input, during the “id_rst” condition the g2_ini and rd_en which is an enable signal will be assigned with Zeros.






When the id_rst is low then rd_en will set to high and based on the PRN Id, the initial condition will be assigned at the output, later it will be passed to the next module i.e. PRN Sequence generator.  The code is attached to Annexure_I

PRN Sequence Generator Module
This module generates the PRN Sequence based on the selected id using the shift registers and XOR gates, based on the polynomial the XOR operations will be done.    The sequence will be generated based on the initial conditions of the registers. 
I/O Ports:
Input
clk: Clock input to the design which supplies 50Mhz which is 20ns.
rst: The reset here is a active high reset which activates on the active high signal.
rd_ene: rd_ene signal input will be received from the PRN_id selector module, goes high when reset goes low.
g2_in: The initial condition of the G2 shift register will be received from the PRN_id selector module based on the selected id.

Output
out: From this output the we will get the PRN Sequence which will be 1023 bit long.






When rst goes high the shift registers will be initialized with Zeros and count register will set to One and flag register will set to Zero. When rst condition is set to Low, the rd_ene will be set to High. The g1and g2 shift registers will be initialized to their required initial conditions and flag register will set to high as the condition in the else if block is satisfied. The sequence will be generated based on the initial conditions of the registers. 



At the next clock cycle the else if condition which contains the flag and rst condition will be satisfied from here the generation of actual sequence will starts. The count condition will be checked for every next clock cycle until the count value reaches to 1024, then the loop will be exited and registers will be set to their initial conditions.

PRN Sequence Generator Top Module
In this top module the other two modules will be integrated with the required port connections.
I/O Ports:
Inputs
CLK: The common clock port for the two modules PRN_ID Selector and PRN_Sequence Generator. The clock will be 20ns clock.
RST: The common reset connection for two modules PRN_ID Selector and PRN_Sequence Generator, it will be active high synchronous reset.
PRN_ID: In this is the port the Id will be ranging from 0-15, we have access from id 1-14.
OUTPUT
RD_ENE:  signal input will be received from the PRN_id selector module, goes high when reset goes low. Then the initial conditions will be loaded into the register.
PRN_Seq_out: This output produces 1023-bit long sequence based on the id we are selecting.

After integrating the all the modules in the top module, this will be the Elaborated Design.







After writing those modules they will be integrated in the top module the output of the first module such as PRN Id and rd_ene will be passed as an input to the second module and other ports which are Clock, Reset, RD_ENE and PRN_Seq_out will be integrated to the top module.

As we can see that rd_en and g2_ini is passing from PRN_ID module to the PRN_SEQ_GEN module. The CLK and RST will be common ports and the reset which is used here is Synchronous Reset which triggers only at the posedge edge of the CLK and the design will operate only at the posedge of the CLK.
